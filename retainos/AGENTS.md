# RetainOS Project Memory

Status: Archived / Killed on 2026-03-10

This file is the live memory for the RetainOS project.

RetainOS is no longer an active build.

## Final Verdict

RetainOS as a standalone hosted cancellation-flow company is a no-go.

Reason:

The obvious wedge is already too crowded by Stripe, RevenueCat, Paddle Retain, Chargebee Retention, Churnkey, and similar tools.

## Company

Name: RetainOS

Category considered: subscription monetization control plane

Current scope: archived

## Original Thesis

RetainOS was intended to own one narrow revenue moment for web-first subscription businesses:

the user clicks `Cancel`

At that moment, RetainOS would run a hosted save flow, test approved variants, measure long-horizon outcome quality, and keep only the winners.

## Original Core Metric

`30-day net retained revenue per cancel-intent user`

Plain English:

Among users who start the cancel flow, how much real revenue is still alive 30 days later because of the system.

## Why The Idea Was Killed

- hosted cancel flows are already common
- adaptive offers already exist
- retention coupons and smart routing already exist
- reaching parity would likely waste a year

## Competitor Reality

Validated competitors and adjacent players:

- Churnkey
- Paddle Retain
- Stripe cancellation flows
- Chargebee Retention
- RevenueCat Customer Center
- KeepFlow

## Churnkey Read

Important finding:

Churnkey Adaptive Offers already optimizes discount percentage and duration inside cancel flows and claims strong retention and LTV improvements.

Conclusion:

The shallow AI-offer wedge is already occupied.

## What Still Might Be Interesting Later

Only revisit if interviews prove a real gap in:

- vendor-independent control layer
- long-horizon evaluator quality
- enterprise policy and trust layer
- cross-vendor experiment memory

## What We Are Not Doing

- no MVP
- no pilot outreach
- no hosted cancel-flow build
- no more execution time on this concept in its current form

## End State

RetainOS is preserved as research, not as an active company build.
